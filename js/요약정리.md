## 함수
- 함수(파선아실)
    * 여기서 x, y를 보통 한국에서는 인자
    * 매개변수(파라미터, parameter) : x, y
    * 전달인자(아규먼트, argument) : 3, 5
    * 사용이유
        1. 재사용성
        2. 아키텍처 파악
        3. 유지보수
        ```js
        function 땅다지기(){}
        function 시멘트작업(){}
        function 철근(){}
        function 벽돌(){}
        function 지붕(){}
        땅다지기()
        시멘트작업()
        철근()
        벽돌()
        지붕()
        ```
    * 예제
    ```js
    function add(x, y){
        return x + y;
    }

    add(3, 5)

    function add(a = 100, b = 200) {
        return a + b;
    }

    add(10, 20);
    // 30
    add(10);
    // 210
    add();
    // 300
    add(b=300)
    // 500
    add(undefined, 300);
    // 400

    function add({ a = 100, b = 200 }) {
        console.log(a+b);
    }

    add({b: 300});
    ```

- 콜백함수
    ```js
    function add(x, y) {
        return x + y;
    }

    function mul(x, y) {
        return x * y;
    }

    function cal(a, b){
        return a(10, 10) + b(10, 10);
    }

    cal(add, mul);
    ```


- 화살표함수
    ```js
    function add(x, y) {
        return x + y
    }

    let addArrow = (x, y) => x + y;
    ```

- 기명 함수
    ```js
    // 기명 함수
    let aa = function sum(x, y) {

    // 익명 함수인것 같지만 바뀜
    ```


## 리터럴
* 리터럴은 코드에 보여지는 타입의 표현을 의미
* 예를 들어 원래 배열을 생성하려면
new Array(4,5,6); 이렇게 해야하지만 [4,5,6] 이렇게 만들수도 있는데요. 이렇게 배열을 의미하는 코드( [4,5,6] )를 배열의 리터럴 표현이라고 합니다.
* 마찬가지로 new Object() 의 리터럴 표현이 {}, new Number(5) 의 리터럴 표현이 5, new String("hello") 의 리터럴 표현이 "hello" 입니다.
